Simulation models and scenarios can be verified or validated with the help of various kinds of integrity constraints:

  1. The OESjs simulator has a number of built-in property and model constraints, which are checked automatically or on demand:

    1. Many simulator objects, such as sim.model, sim.scenario and sim.config, have pre-defined property constraints that allow validating their property values set in the simulation.js file.
    2. For any particular class of simulation models, e.g., for Processing Network models, the OESjs simulator may have pre-defined model constraints that can be checked at each simulation step for verifying certain model correctness properties.

  2. A simulation developer can define property constraints within the definition of object and event types for allowing the simulator to verify that the values of properties are valid.

  3. A simulation developer can define model constraints for allowing the simulator to verify that the model satisfies certain correctness properties.

Constraint checking is useful in the development of new simulation models, since it allows to catch a number of errors a developer may make. When a simulation has been sufficiently tested and is to be deployed for operational usage, constraint checking will normally be turned off in favor of simulation execution speed.
