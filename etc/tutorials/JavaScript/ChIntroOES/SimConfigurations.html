A simulation scenario can be configured in various ways using, for instance, different types of visualizations and various user interfaces (UI):

  1. Turn off multi-threading (which is normally used for speeding up simulation execution) by setting the configuration parameter sim.config.runInMainThread to true (it is false by default). Running the simulation in the main thread, together with the user interface, may simplify debugging.
  2. Turn on/off the simulation log by setting the configuration parameter sim.config.createLog to true/false.
  3. Suppress or show the initial state UI, which allows to inspect/modify the initial values of model variables and the initial state of objects.
  4. Turn on/off visualization, if there is one, by setting the configuration parameter visualize to true/false.
  5. Turn on/off user interaction, if there is one, by setting the configuration parameter userInteractive to true/false. Since user interaction requires visualization, it is also turned off when sim.config.visualize is set to false.
  6. Slow down a (standalone) scenario simulation run by setting the configuration parameter stepDuration, which defines the duration of a simulation step (in ms). This is typically used for being able to observe a simulation run.
  7. Credit art work used in a visualization with the parameter artworkCredits.

In the simulation definition file, we could have settings like the following:

sim.config.createLog = true;
sim.config.suppressInitialStateUI = true;
sim.config.visualize = true;
sim.config.userInteractive = false;
sim.config.stepDuration = 200;   // 200 ms observation time per step
sim.config.artworkCredits = "Weather icons by https://icons8.com";
