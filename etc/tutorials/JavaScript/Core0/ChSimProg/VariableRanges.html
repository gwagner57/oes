OESjs is based on the cLASSjs library, which supports choosing specific datatypes, such as "PositiveInteger", as the range of model variables and object/event properties. In addition, various constraints can be defined for them. Defining specific ranges helps catching type errors and providing tailored user interfaces. Defining other constraints helps catching other simulation programming flaws.

Range Constraints

Suitable range constraints can be defined by using one of the following datatype keywords:

  • "String", "NonEmptyString", "Identifier" (disallowing blank spaces and other special characters), "Email", "URL", "PhoneNumber".
  • "Integer", "PositiveInteger", "NonNegativeInteger".
  • "Decimal", "Number" (the same as "Decimal"), "Percent", "ClosedUnitInterval" [0,1], "OpenUnitInterval" (0,1).
  • "Boolean".
  • "DateTime", "Date" (a JS Date value or an ISO date or date-time string).

For a decimal number type, the number of decimal places to be displayed can be specified with a declaration parameter setting like displayDecimalPlaces:2, as shown in the following example 

var StaffMember = new cLASS({
  Name: "StaffMember",
  supertypeName: "oBJECT",
  properties: {
    "age": { range: "PositiveInteger"}
    "weight": { range: "Decimal", displayDecimalPlaces: 2}
  }
});

In addition to the simple/primitive datatypes listed above, the range of a property may be

  1. the name (or the corresponding object variable) of an enumeration, or
  2. the name of another class (defined as an instance of cLASS).

Optional Properties and Multiplicity Constraints

A variable or property may be optional or multi-valued. By default, variables and properties are mandatory and single-valued.

An example of an optional single-valued property is clerk, while an example of an optional multi-valued property is waitingCustomers, defined as follows:

var ServiceDesk = new cLASS({
  Name: "ServiceDesk",
  supertypeName: "oBJECT",
  properties: {
    "clerk": { range: "StaffMember", optional: true}
    "waitingCustomers": { range: "Customer", minCard: 0, maxCard: Infinity}
  }
});

Notice how a (by default) single-valued property like clerk is defined to be optional with the help of the declaration parameter setting optional:true, while a multi-valued property like waitingCustomers is defined to be optional with minCard:0 and multi-valued with maxCard:Infinity.
