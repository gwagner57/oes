An organizational position is defined by a name and a set of organizational role types, which are performer types representing the responsibilities of the position holders. Recall that a performer is a special type of resource object required for performing activities of a certain type. For instance, in a medical department, the organizational position Nurse may be defined by the set of performer types Guide and ExaminationAssistant, where a Guide is required for guiding a patient to an examination room and an ExaminationAssistant is required for assisting a doctor in a medical examination.

Computationally, an organizational position is an object type that is a subtype of one or more performer types, implying that its instances (position holders) are resources that can be chosen for performing the corresponding activities.

For instance, the organizational position Nurse is a subtype of the performer type Guide and of the performer type ExaminationAssistant, with the meaning that nurses play both the role of guides and the role of examination assistants.

In simple cases, an organizational position is defined by just one performer type. In such a case, the organizational position is identified with the performer type

In an activity-based Discrete Event simulation, each organizational position defines a resource pool (or maybe better: performer pool) that includes all employees holding that position.

Resource Pools Assigned to Multiple Activity Types

When a resource pool represents an organizational position defined by n performer types, it is shared among all n corresponding activity types.
