This minimal Activity Network model consists of two nodes: an initial event node for recurrent "CustomerArrival" events and an activity node for "Service" activities, which require a service desk as a (performer) resource. These two nodes are connected with a Resource-Dependent Activity Scheduling (RDAS) arrow.
